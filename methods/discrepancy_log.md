## 1. なぜ乖離を記録するか

### 乖離記録の目的

OOπ・TTT・E式の理論は、完成した体系ではない。検証と更新を繰り返すことで育っていく「生きた理論」である。

その育ちの主な燃料は**乖離の記録**である。

```
当たった記録が教えること：
  「この変数の組み合わせが、このケースでは機能した」
  → 「機能した」という事実
  → しかし「なぜ機能したか」は当たった記録だけでは見えにくい

外れた記録が教えること：
  「この変数の組み合わせが、このケースでは機能しなかった」
  → 「機能しなかった理由」が欠落変数を指し示す
  → 理論の「盲点」を明示的に示してくれる
```

気象予報の世界でも、外れた予報の事後分析（検証）が予報精度を向上させてきた。OOπ理論も同じ原理で育てる。

### 「失敗の記録」ではなく「発見の記録」

乖離記録は「予想が間違った」という失敗の記録ではない。

```
正しい見方：
  乖離 = 「この領域には、まだ自分が見えていない変数がある」
         という情報

乖離記録の価値：
  乖離が多いほど、発見の機会が多い
  乖離が深いほど、重要な変数が欠落している
  乖離のパターンが繰り返されるほど、
  構造的な欠落（第7項以降の変数）が存在する
```

## 2. 乖離の四分類と診断法

### 分類①：量的乖離（大きさが違う）

```
定義：
  予想した変形の方向は正しかったが、
  規模（大きさ）が予想と違った

例：
  「大雨になる」は当たったが、
  「100mmの降水」と予想したのに「300mm」降った

診断：
  → どの変数の係数が誤っていたか？
  → vVの係数を過小評価していたか？
  → hHによる増幅を考慮していなかったか？
  → 正しい方向で「距離感」だけが違う乖離

対処：
  係数の修正で対応できることが多い
  または hH（前歴による増幅）の有無を確認
```

### 分類②：位置的乖離（場所が違う）

```
定義：
  予想した変形の大きさは近かったが、
  変形が起きた場所（領域）が予想と違った

例：
  「山形県全体で大雨」と予想したが、
  実際は「真室川町の山間部に集中」した

診断：
  → sS（スケール）の設定が適切でなかったか？
  → zZの地形変換を見落としていたか？
  → eE（地形・地理的条件）を無視していたか？
  → 「何が起きるか」は当たったが「どこで」が違う乖離

対処：
  sS（空間スケール）の調整
  zZの地形変換係数の追加
  eE（境界条件：地形）の明示的な組み込み
```

### 分類③：時間的乖離（タイミングが違う）

```
定義：
  予想した変形の内容は近かったが、
  変形が起きたタイミング（時刻・時期）が違った

例：
  「夜間に前線が活発化する」と予想したが、
  「翌朝まで遅れた」

診断：
  → Uu（変化率）の読み誤りか？
  → nN（臨界）の到達タイミングの予測精度か？
  → cC（連鎖）のタイムラグを考慮していなかったか？
  → 「何が起きるか」は当たったが「いつ」が違う乖離

対処：
  Uu（変化率）の詳細な追跡
  nNの到達条件の精緻化
  Uuの二重構造（短期・長期）の区別
```

### 分類④：質的乖離（現象の種類が違う）

```
定義：
  予想した変形の「種類」が違った。
  起きたことが予想とは別の「カテゴリー」の現象だった

例：
  「大雨が降る」と予想したが
  「被害の規模が想定外に大きかった」（質が違う）
  
  「組織が安定する」と予想したが
  「組織が崩壊した」

診断：
  → 最も重要な乖離。P式に入っていない変数が存在するサイン。
  → 既存変数のどの組み合わせでも説明できないか確認する
  → 説明できなければ新変数の候補

対処：
  新変数の探索（理論の更新）
  第7項以降の変数候補の検討
  複数の独立した事例での確認
```

## 3. 乖離記録の書き方

### 記録の原則

```
原則①：具体的に書く
  「外れた」ではなく
  「○○を予想したが○○が起きた。
   差は○○という点にある」と書く

原則②：乖離の種類を特定する
  量的・位置的・時間的・質的のどれかを明示する
  複数に当てはまる場合は全て記録する

原則③：欠落変数の候補を書く
  「なぜこの乖離が起きたか」の仮説として
  欠落していた変数の候補を書く

原則④：次回の観測への反映を書く
  この乖離から何を学んで
  次回のフライトプランに何を追加するかを書く
```

### 乖離記録の書式

```markdown
## 乖離記録 #[番号]

### 基本情報
記録日：
領域：
事例の概要：

### 予想と実際の対比

| 項目 | 予想（フライトプラン） | 実際に起きたこと |
|------|---------------------|----------------|
| 変形の内容 | | |
| 変形の規模 | | |
| 変形の場所 | | |
| 変形のタイミング | | |

### 乖離の分類
- [ ] 量的乖離（規模が違う）
- [ ] 位置的乖離（場所が違う）
- [ ] 時間的乖離（タイミングが違う）
- [ ] 質的乖離（種類が違う）← 最重要

### 乖離の詳細

乖離が生じた原因の仮説：

既存変数（xX〜wW）で説明できるか：
  説明できる → どの変数の係数が問題だったか：
  説明できない → 新変数の候補：

### 欠落変数の候補

候補変数名：
定義：
気象での対応（あれば）：
社会・組織での対応（あれば）：
他の事例での確認（あれば）：

### 第7項以降の変数候補としての評価

複数事例での繰り返し確認：  有・無・未確認
有無だけで予測精度が変わるか：  変わる・変わらない・不明
既存変数との独立性：  独立・重複・不明

追加判定：  確定・要検討・候補・棄却

### 次回のフライトプランへの反映

STEP 3（変数の特定）に追加するチェックリスト：
STEP 4（iI推定）に追加する視点：
STEP 5（フライトプランの設計）に追加するシナリオ：
```

## 4. 蓄積から見えてくるパターン

### パターン分析の目的

個々の乖離を記録するだけでなく、蓄積した記録から**繰り返し現れるパターン**を発見することが、理論の最大の育ちの源泉となる。

```
パターンが示すもの：
  「この領域では、いつも○○が欠落する」
  → 構造的な欠落変数の存在

  「この種類の乖離が繰り返される」
  → P式の特定の次元が系統的に弱い

  「A領域とB領域で同じパターンの乖離がある」
  → その乖離を説明する変数が普遍的に存在する
```

### パターン分析の実施方法

**定期的な棚卸し（3ヶ月〜6ヶ月ごと）：**

```
Step 1：蓄積した乖離記録を並べる
  時系列・領域・乖離の種類で整理する

Step 2：繰り返しパターンを探す
  同じ種類の乖離が3回以上現れていないか
  同じ変数が欠落していないか
  同じ領域で同じパターンが繰り返されていないか

Step 3：「理論の盲点地図」を描く
  どの領域のどの種類の変形で、
  どの変数が系統的に欠落しているか

Step 4：変数の追加・修正の判断
  パターンが3事例以上で確認されれば
  第7項以降の変数として追加を検討する
```

### このリポジトリの検証から見えたパターン

現在までの検証（気象・司法・社会OS）から見えてきたパターン：

```
パターン①：hH（前歴）の系統的欠落
  気象（能登：地震後の地盤脆弱化）
  司法（砂川事件以来の統治行為論）
  社会OS（旧OSの第二・三層への残存）
  → 複数領域で確認 → 確定変数として採用

パターン②：zZの係数過大評価
  「制度上はzZがある」と見るが
  実態はzZ≒0である
  気象：地形変換係数の無視
  司法：統治行為論・付随的審査制
  → 「制度」と「実効」の差を常に確認する必要性

パターン③：dD（散逸・抵抗）の無視
  「なぜ変わらないか」の説明に
  dDが欠落していることが多い
  → 「変わる力（vV・Uu）」だけでなく
    「変わらない力（dD）」を常にセットで見る

パターン④：eEによるzZ上書き
  外部からの圧力（占領・米国・市場）が
  独立しているはずのzZを上書きする
  → jJ（eE×zZの相互作用）の確認が必要
```

## 5. 乖離から変数を発見する方法

### 発見のプロセス

能登大雨でhH（前歴）変数が発見されたプロセスを詳細に記録する。これが変数発見の「標準的なプロセス」の実例である。

```
観測（2024年9月）：
  気象変数は正確に把握できていた
  「大雨になる」という予報は当たった

乖離の発見：
  「被害規模」が気象予報の想定を大幅に超えた
  → 質的乖離（現象の種類が違う）として記録

乖離の分析：
  「大雨 → 大被害」は気象変数で説明できる
  しかし「なぜこれほどの被害か」は説明できない
  → 既存変数（xX〜wW）では説明が足りない

欠落変数の候補探索：
  「地震後の地盤脆弱化」があれば説明できる
  → 「同じ降水量でも地盤の状態によって被害が違う」
  → これは「系の前状態が応答を変える」という概念

新変数の定式化：
  hH（前歴・履歴）：
  P_応答 = P_入力 × f(hH)
  f > 1 で脆弱化、f < 1 で強化

他領域での検証：
  司法：砂川事件（1959年）というhHが
       現在のzZ機能不全を規定している
  社会OS：旧OSのhHが第二層として残存し
          制度変更（rR変更）を無効化する

結論：
  hHは複数の独立した領域で確認された
  有無を判断するだけで予測が変わる
  既存変数と独立している
  → 確定変数として theory/04_variables.md に追加
```

### 新変数の評価フレームワーク

```
評価項目①：複数事例での繰り返し確認
  1事例：偶発的な可能性が高い
  2事例：注目に値する
  3事例以上：構造的な欠落の可能性が高い
  異なる領域での確認：普遍的な変数の可能性

評価項目②：有無だけで予測精度が変わるか
  正確な値がわからなくても
  「有」と「無」で予想の方向性が変わるなら
  実践的価値がある

評価項目③：既存変数との独立性
  既存変数の係数変更で代替できる場合は
  新変数として追加しない
  （係数の修正で対応する）

評価項目④：命名の適切さ
  2文字で表現できる意味を持つか
  気象と社会の両方に対応する解釈があるか
```

## 7. 乖離記録のメンテナンス

### 記録の鮮度管理

```
記録直後（照合後すぐ）：
  乖離の事実と種類の記録
  欠落変数の仮説の記録

1週間後：
  欠落変数の候補を他の事例で確認できるか検索
  「理論の盲点地図」の更新

1ヶ月後：
  パターン分析の更新
  新変数の追加判断

6ヶ月後：
  全乖離記録の棚卸し
  theory/04_variables.md の更新
  フライトプランの設計改善への反映
```

### Issues への投稿

このリポジトリのIssuesに乖離記録を投稿することで、他の領域の観察者と共有できる。

```
Issues 投稿の書式：

タイトル：[乖離記録] [領域名] 乖離の概要

本文：
- 観察した表面：
- 予想した裏面：
- 実際に起きたこと：
- 乖離の種類（量的・位置的・時間的・質的）：
- 欠落変数の候補：
- 他の領域での確認（あれば）：
```
